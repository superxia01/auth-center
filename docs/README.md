## 🎯 使用指南

### 对于新业务系统

1. **第一步**: 阅读 `ARCHITECTURE_OVERVIEW.md`
   - 理解 V3.0 架构标准
   - 选择技术栈（Go + Next.js）
   - 设计数据库模型

2. **第二步**: 阅读 `AUTH_CENTER_INTEGRATION_GUIDE.md`
   - 集成账号中心统一认证
   - 实现三层账号模型
   - 参考已集成系统案例

3. **第三步**: 参考 `pr-business` 仓库
   - 查看完整的前后端分离实现
   - 复用代码模式
   - 遵循相同的目录结构

### 对于现有系统迁移

1. 评估当前架构（V1.0/V2.0）
2. 制定迁移计划（参考 V3.0 标准）
3. 前后端拆分：
   - 前端：保留 UI 组件，移除 API Routes
   - 后端：使用 Go + Gin 重写 API
4. 集成账号中心认证
